## Problem

E-commerce stores bleed revenue at the consideration stage. A customer lands on a product page, has a specific question about stock availability, sizing, or how two variants compare, and finds nothing useful. Static FAQ pages are generic by design. Live chat requires scheduling human agents around the clock. Most stores have neither the budget nor the staffing to cover every question in real time, so the customer leaves.

The gap is not a content problem — the product data already exists inside the catalog. The gap is access. Customers need a way to query that data conversationally, instantly, without waiting for a human.

## Approach

TechCurator solves this with a floating chat widget backed by a LangChain agent that is grounded entirely in the store's product catalog. At setup, the catalog is ingested, chunked, and embedded into a vector store. From that point forward, every customer question — stock availability, product specs, size guides, cross-product comparisons — is answered by retrieving the relevant catalog data and generating a response in real time.

Checkout guidance uses structured response templates that mirror the store's purchase flow, so the agent does not just answer questions but actively moves customers toward a decision. The entire widget embeds in any storefront via a single script tag, requiring no framework-specific integration work on the merchant's side.

The agent is stateful within a session, so follow-up questions resolve correctly without the customer repeating context. When a question falls outside catalog scope, the agent acknowledges this cleanly rather than hallucinating an answer.

## Stack

- **LangChain** — agent framework for catalog-grounded question answering with retrieval-augmented generation.
- **FastAPI** — async backend that handles catalog ingestion endpoints and streams chat responses to the widget.
- **Python** — core runtime powering the catalog processing pipeline, chunking logic, and embedding generation.
- **Chatbot Development** — floating widget UI built with streaming response support so answers appear token by token.
- **API Integration** — product catalog sync layer and storefront embed delivered via a single script tag drop-in.
